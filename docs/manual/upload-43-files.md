# คู่มือการ Upload ข้อมูล 43 แฟ้ม (Health Data Center: HDC)

เอกสารฉบับนี้อธิบายขั้นตอนการอัปโหลดไฟล์ข้อมูลมาตรฐาน 43 แฟ้มเข้าสู่ระบบ HDC สำหรับหน่วยบริการ

---

## 1. การเตรียมไฟล์ข้อมูล (File Naming Convention)

!!! danger "ข้อกำหนดที่ต้องปฏิบัติตามอย่างเคร่งครัด"
    ชื่อไฟล์ที่ผู้ใช้งานอัปโหลด **จะต้องตรงกับรหัสหน่วยบริการ (hospcode)** ของผู้ใช้งานที่กำลัง Login อยู่เท่านั้น จึงจะสามารถอัปโหลดได้

### รูปแบบการตั้งชื่อไฟล์
ใช้โครงสร้างชื่อไฟล์ดังนี้: `F43_xxxxx_YYYYMMDDHHmmss.zip`

* **xxxxx**: รหัสสถานบริการ 5 หลัก
* **YYYYMMDDHHmmss**: ปี เดือน วัน และเวลาที่ส่งข้อมูล

!!! example "ตัวอย่างชื่อไฟล์"
    `F43_41124_20250325144500.zip`

---

## 2. ขั้นตอนการอัปโหลดข้อมูล

### เข้าสู่เมนูการนำเข้าข้อมูล
1. ไปที่แถบเมนูหลัก **"นำเข้า/ส่งออกข้อมูล"**
2. เลือกเมนูย่อย **"อัปโหลดข้อมูล 43 แฟ้ม"**

### การเลือกไฟล์และอัปโหลด
1. คลิกปุ่ม **"Click to Upload Files"** เพื่อเลือกไฟล์ `.zip`
2. ระบบจะแสดงสถานะการอัปโหลดในตาราง **"สถานะการอัปโหลดข้อมูล"**

!!! info "การตรวจสอบเบื้องต้น"
    หลังการอัปโหลด ระบบจะแสดงชื่อไฟล์ ขนาดไฟล์ (Size) และความคืบหน้า (Progress) ให้คุณทราบทันที

---

## 3. การตรวจสอบสถานะการนำเข้า

คุณสามารถติดตามผลได้ที่ตาราง **"ประวัติการอัปโหลด"**

| หัวข้อ | รายละเอียด |
| :--- | :--- |
| **สถานะ PROCESS** | แสดงความคืบหน้าของระบบ |
| **สถานะ FINISH** | ระบบประมวลผลไฟล์เสร็จสมบูรณ์ |
| **ขนาดไฟล์** | ขนาดของไฟล์ที่อัปโหลด (เช่น 811.29 MB) |
| **วันที่** | วันและเวลาที่อัปโหลดข้อมูล |

---

## 4. ผลการนำเข้าข้อมูล 43 แฟ้ม

ไปที่เมนู **"ผลการนำเข้าข้อมูล 43 แฟ้ม"** เพื่อตรวจสอบรายละเอียดรายแฟ้ม (เช่น `accident.txt`, `surveillance.txt`)

!!! success "ข้อมูลที่ปรากฏในรายงาน"
    * **ส่งข้อมูลสำเร็จ**: จำนวนระเบียนที่นำเข้าได้
    * **ข้อมูล Error**: จำนวนระเบียนที่มีปัญหาและไม่เข้าสู่ระบบ
    * **วันเดือนปีที่นำเข้า**: วันที่ระบบประมวลผลข้อมูล
